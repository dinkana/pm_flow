# Project Health Check

**Быстрая диагностика проекта для ПМ / БА**

## Что это

Инструмент для самодиагностики проектов. За 12 вопросов даёт объективную оценку состояния проекта и конкретный план корректирующих действий.

**Без регистрации. Без сервера. Работает оффлайн.**

## Для кого

- **Руководители команд** (Team Lead, Tech Lead, Product Owner) — быстро оценить состояние проекта
- **Основатели стартапов** — без выделенного менеджера проектов понять, где проблемы
- **Фрилансеры** — проверить несколько проектов одновременно
- **Junior PM/BA** — подготовиться к встрече с заказчиком

## Как использовать

1. Откройте [dinkana.github.io/pm_flow](https://dinkana.github.io/pm_flow/)
2. Ответьте на 12 вопросов (по 2 на каждую область)
3. Получите оценку здоровья проекта и план действий
4. Скачайте PDF или скопируйте отчёт

## Что оценивается

6 ключевых областей:
- **Цели и границы** — понимает ли команда, зачем проект
- **Стейкхолдеры** — осознанная работа с заинтересованными лицами
- **Планирование** — реалистичные прогнозы на основе данных
- **Риски** — системная работа с неопределённостью
- **Команда** — прозрачность процессов и дисциплина
- **Качество** — обратная связь от пользователей

## Результаты

- **Общий индекс здоровья** (0–100%) с цветовой индикацией
- **Разбивка по областям** — видно, где просадка
- **Приоритетные действия** — конкретные шаги с измеримым результатом
- **Экспорт** — PDF, Markdown, текст

## Методология

Вопросы базируются на PMBOK и Agile-принципах. Каждый вопрос — поведенческий индикатор, а не проверка формальных артефактов.

**Шкала ответов:**
- Да (3 балла) — практика устоялась
- Скорее да (2 балла) — выполняется нерегулярно
- Скорее нет (1 балл) — осознаётся как проблема
- Нет (0 баллов) — отсутствует

**Формула здоровья:**
```
Health = avg(Scope×1.3, Stakeholders, Planning, Risks, Team, Quality)
```

Область "Цели и границы" имеет коэффициент 1.3 как критичная.
